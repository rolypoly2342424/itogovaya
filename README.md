# Пораждающий:
Проект реализует паттерн **Абстрактная фабрика** для создания кроссплатформенных элементов пользовательского интерфейса (UI).

## Описание изменений (Branch: feature/abstract_factory_ui)
В рамках данного обновления были реализованы следующие изменения:
*   **Поддержка Linux**: Добавлена новая конкретная фабрика LinuxFactory и соответствующие продукты.
*   **Интерактивность**: В интерфейсы продуктов добавлен метод interact() для имитации действий пользователя.
*   **Новый компонент**: В общее семейство UI-элементов добавлен CheckBox.
*   **Безопасность**: Добавлены виртуальные деструкторы для корректного управления памятью.

## Структура семейств UI
Система разделена на два независимых семейства продуктов. Каждое семейство гарантирует, что все созданные элементы будут визуально и функционально совместимы друг с другом.

### 1. Семейство Windows
*   **WinButton**: Кнопка с характерным поведением Windows.
*   **WinScrollBar**: Полоса прокрутки для ОС Windows.
*   **WinCheckBox**: Флажок (чекбокс) в стиле Windows.

### 2. Семейство Linux
*   **LinuxButton**: Кнопка, адаптированная под Linux-системы.
*   **LinuxScrollBar**: Специфичный для Linux элемент прокрутки.
*   **LinuxCheckBox**: Чекбокс с логикой Linux.

## Техническая реализация

Клиентский код взаимодействует только с абстрактными интерфейсами:
1.  UIFactory — интерфейс для создания продуктов.
2.  Button, ScrollBar, CheckBox — интерфейсы продуктов.

**Пример логики выбора фабрики:**
```cpp
string config = "linux"; // Выбор платформы
UIFactory* factory;

if (config == "win") factory = new WinFactory();
else if (config == "linux") factory = new LinuxFactory();
---

# Структурный: Decorator UI

Проект реализует структурный паттерн **Декоратор**, который позволяет динамически добавлять новые функциональные возможности объектам, не изменяя их исходный код.

## Описание изменений (Branch: feature/ui_decorators)

В рамках данного обновления реализована система гибкой настройки внешнего вида UI-элементов:
*   **Base Component**: Создан абстрактный интерфейс `Widget` для всех графических элементов.
*   **Concrete Component**: Реализована базовая `Button`, имеющая только стандартную отрисовку.
*   **Decorator Base**: Добавлен базовый класс-обертка `WidgetDecorator`.
*   **Concrete Decorators**: Созданы декораторы `BorderDecorator` (рамка) и `ShadowDecorator` (тень).

## Структура компонентов

Система позволяет "оборачивать" один декоратор в другой, создавая сложные визуальные эффекты путем композиции.

### 1. Базовые элементы
*   **Button**: Простой текстовый элемент управления.
*   **TextField**: Поле ввода текста (базовый компонент).

### 2. Визуальные декораторы (Обертки)
*   **BorderDecorator**: Добавляет цветную рамку вокруг любого виджета.
*   **ShadowDecorator**: Добавляет эффект тени для создания объема.
*   **ScrollDecorator**: Добавляет полосы прокрутки, если контент не помещается.

## 🛠 Техническая реализация

Декоратор наследует интерфейс компонента и одновременно содержит объект этого интерфейса внутри себя.

### Ключевые преимущества:
1.  **Гибкость**: Можно комбинировать эффекты в любом порядке (например, кнопка с тенью в рамке).
2.  **Чистота кода**: Класс `Button` не раздувается методами отрисовки теней и рамок.
3.  **Принцип открытости/закрытости**: Новые визуальные эффекты добавляются созданием новых классов-декораторов.

### Пример сборки "слоеного" объекта:
```cpp
// Создаем базовую кнопку
Widget* myButton = new Button("Ок");

// Оборачиваем её в рамку
myButton = new BorderDecorator(myButton, "Черная");

// Добавляем поверх рамки тень
myButton = new ShadowDecorator(myButton);

// Один вызов отрисует всю цепочку эффектов
myButton->draw(); 
// Создание сканера через адаптер объекта
OldLPTScanner oldScanner;
USBScanner* scanner = new LPTToUSBScannerAdapter(&oldScanner);
scanner->scanUSB(600);
